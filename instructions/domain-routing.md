# Domain Routing

- Load exactly one primary domain skill before making implementation or planning decisions for specialized tasks.
- Use `fpga` for VHDL, Verilog, SystemVerilog, Vivado, timing, CDC, AXI, and XDC work.
- Use `c-systems` for generic C or C++ systems code, low-level interfaces, bitwise logic, buffers, drivers, and portability concerns.
- Use `vitis` for Xilinx BSP, bare-metal runtime, DMA, interrupt, cache coherency, PS-PL interaction, and linker-script-adjacent work.
- Use `python` for Python application, tooling, automation, packaging, and pytest work.
- Use `dotnet` for C#, .NET, MVVM, ASP.NET, EF Core, dependency injection, and async/await work.
- Use `document` for Markdown, README, changelog, or technical writing tasks.
- Use `ai-toolkit` for OpenCode config, self-hosted model routing, provider selection, prompt design, and coding-assistant tuning.
- If a task spans multiple domains, choose the primary implementation domain first and only layer `document` or `ai-toolkit` when the task is explicitly about those areas.
