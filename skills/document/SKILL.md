# Document

## When to use me

Use this skill for:

- README changes
- changelogs
- technical guides
- migration notes
- config and setup documentation

## What I enforce

- preserve existing structure unless the user asks for a rewrite
- avoid deleting information under the label of cleanup
- keep headings, tables, code fences, and lists stable
- prefer targeted edits over full rewrites

## Critical checks

- keep commands copy-pasteable
- keep file paths and config keys exact
- note assumptions instead of inventing missing project facts
